# Memory Guidance

Distill signal, not chatter.

- Keep `MEMORY.md` under 200 lines.
- Write raw session notes to `sessions/YYYY-MM-DD.md`.
- Curate only stable truths into `MEMORY.md`.
- Prefer decisions over discussion.
- Prune stale assumptions when the live system changes.

For audio work, prioritize remembering:

- mix rules and loudness decisions
- agreed asset naming conventions
- fallback strategy for missing sounds
- dialog voice sourcing constraints
- known noisy or fatiguing event paths
