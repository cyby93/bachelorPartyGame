## PixelLab MCP findings (read fresh 2026-04-17)
- `create_map_object`: minimum 32px canvas. Only viable tool for standalone effect sprites.
- `create_character`: character rigs only, not useful for VFX/projectiles.
- `create_tiles_pro`, `create_isometric_tile`, `create_topdown_tileset`, `create_sidescroller_tileset`: tileset tools — wrong for isolated effect sprites.
- **16×16 projectiles are not possible via PixelLab MCP.** Workaround: generate at 32×32 and render at 16×16 in engine (at projectile travel speed the softness is invisible).
- Effects section in `docs/PIXELLAB.md` is **TBD** — no style parameters locked yet. Need to define before bulk effect generation.

## SpriteKey system design
- `spriteKey` is optional in SkillDatabase. Server passes it through projectile state. Client reads it in ProjectileSprite.js, falls back to `projectile_default`.
- Custom sprites use `tint = 0xffffff` (no class color wash). Default sprite keeps class color tint.
- AuraSystem: shield effect renders `aura_pw_shield` Sprite instead of a Graphics ring. Sized at `(baseRadius + 10) * 2` to wrap around character.

## What stays as Graphics (do not sprite yet)
- `meleeArc` — geometric, parameterized, sprite can't replicate.
- `aoeCircle`, `groundZone` — radius-driven, math-dependent.
- `dashTrail` — positional/continuous.
- `buffRing` (generic) — fine until a specific buff needs identity.
- `impactFlash` — catch-all fallback, keep it.

## Rule
Sprite when Graphics "communicates the wrong thing or nothing." Not on a schedule.

## Session — Ability Sprite Pipeline + Projectile Trail System

**What happened:** Built the ability sprite generation pipeline from scratch using PixelLab `create_map_object`, then implemented a full projectile trail system covering two abilities (Avenger's Shield and Penance).

**Key outcomes:**
- Established PixelLab prompt formula for ability sprites: `create_map_object`, `low top-down`, `48×48`, `medium shading`, `single color outline`
- Downloaded `avengers_shield.png` → `public/assets/sprites/abilities/avengers_shield.png`
- Built `TRAIL_STYLE_MAP`, `SPIN_SPEED_MAP`, `BODY_SCALE_MAP`, `TRAIL_LENGTH_MAP` in `ProjectileSprite.js` — all keyed by `spriteKey`, easy to extend per ability
- `holy` trail (Avenger's Shield): 3-layer golden bloom — outer glow + mid ring + white core. Shield spins at 0.14 rad/frame
- `divine` trail (Penance): tighter glow streak + lingering holy sparkles that spawn at the tail (direction-aware, perpendicular scatter only)
- Fixed ricochet bug: chain-spawned projectiles in `SkillSystem.js` were not copying `spriteKey` — one-line fix
- Registered `projectile_avengers_shield` in `HostGame.js` manifest with custom path (abilities subfolder)

**Observations:**
- Cyby's tuning preference: sprite must be dominant — glow/particles should support it, not compete. Start conservative on alpha, go up from there.
- Particle origin matters a lot — spawning in all directions felt wrong; tail-only with perpendicular scatter felt natural immediately
- `low top-down` view for ability sprites gives enough 3D depth without looking flat

**Follow-up:**
- 30 remaining abilities have no sprites or trails — pipeline is now established, ready to scale
- Projectile trail system is extensible — add new styles by registering in the maps and implementing a `_draw{Style}Trail` method
