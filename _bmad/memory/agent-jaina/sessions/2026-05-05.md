# Session 2026-05-05

## Task
Fix ability ordering in `UpgradeSelectScreen.svelte` to match the controller grid layout.

## What Was Done

The controller (`GameScreen.svelte`) renders the 2×2 skill grid using `GRID_ORDER = [1, 3, 0, 2]`:
- Top-left: skill[1], Top-right: skill[3]
- Bottom-left: skill[0], Bottom-right: skill[2]

The upgrade screen (`UpgradeSelectScreen.svelte`) was iterating `skills` in natural array order (0,1,2,3), causing a mismatch.

Fix: added `GRID_ORDER = [1, 3, 0, 2]` and `orderedSkills = $derived(...)` in `UpgradeSelectScreen.svelte`, then changed `{#each skills as skill}` to `{#each orderedSkills as skill}`.

Server sends skills with `skillIndex` property, which is used to map positions correctly via `skills.find(s => s.skillIndex === idx)`.
