# Session 2026-05-02

## Task
KNOWN_ISSUES: host loading/connection time — nothing displayed during slow startup.

## Root Cause
`main.js` calls `await game.init()` before creating the socket or setting up DOM refs. `game.init()` calls `_loadSprites()` which builds a large manifest of all character animation frames + enemy/boss sprites and blocks on `await Assets.load(manifest)`. The badge element existed in HTML but had `opacity: 0` — nothing told it to show until `setConnectionStatus('Connecting...')` ran after init completed. Players saw a static menu with no indication anything was happening.

## Changes Made

### `client/host/HostGame.js`
- `init(containerEl, onProgress = null)` — added optional progress callback parameter
- `_loadSprites(onProgress = null)` — same
- `Assets.load(manifest, onProgress ?? undefined)` — passes callback to PixiJS loader

### `client/host/main.js`
- Before `await game.init()`: grab badge element directly, show "Loading assets… 0%"
- Pass progress callback to `game.init()` that updates badge text with percentage
- After `game.init()` returns: set badge text to "Connecting…" as bridge before socket setup

Existing `setConnectionStatus('Connecting...', 'connecting')` later in module init normalizes classes correctly. `socket.on('connect', ...)` updates to "Connected" with auto-hide as before.

## Result
Badge now shows "Loading assets… N%" from the first tick of sprite loading, "Connecting…" while the socket handshakes, and "Connected" (auto-hides after 2s) once the socket is up.
