### Task 13 — Cooldown/skill button rendering review
**Modified:** `client/controller/components/SkillButton.svelte`

Two issues found and fixed:

**1. Border layout shift on `held`:**
`.skill-btn` default was `border: 0`. The `.held` class adds `border: 2px solid #00d2ff`. Switching from 0 to 2px border caused button contents to shift by 2px on activation.
Fix: `border: 0` → `border: 2px solid transparent` as default.

**2. Dead `class:on-cd` binding with reactivity bug:**
`class:on-cd={expiresAt > Date.now()}` — `Date.now()` is not reactive, so the class would never be removed when a cooldown expires naturally (only removed on next `expiresAt` prop change). Additionally, no CSS existed for `.skill-btn.on-cd`, making the binding a no-op.
Fix: Removed the binding entirely. `CooldownOverlay` handles cooldown visuals via CSS `@property` animation — that's the right layer for this.

**CooldownOverlay.svelte** — no changes needed. The CSS `@property` + conic-gradient approach is correct and efficient.
