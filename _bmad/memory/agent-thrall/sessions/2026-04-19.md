# Session — 2026-04-19

## Topic
Level 2 beam + mirror mechanic design clarification and plan update.

## Clarifications from Cyby

1. **Mirror is a waypoint only** — no deflection math. The beam is a polyline: `[building1.center, mirror.position, building2.center]`. Two straight segments. Mirror has no orientation, no angle, no surface normal.
2. **Mirrors are a base-level config field** — available on any level from Level 1 onward, not Level 2-specific.

## Final Design

### Config contract (both fields optional on every level)

```js
mirrors?: [{ id, position: { x, y } }]

beams?: [{ id, source: buildingId, target: buildingId, waypoints?: [mirrorId] }]
```

### Server resolution

Server resolves beam path at level setup:
```
points = [buildings[source].center, ...waypoints.map(id => mirrors[id].position), buildings[target].center]
```

Sends `{ id, points: [{x,y},...], active }` in delta state. No angle math anywhere.

### Client renderer

Receives `points[]`, draws a polyline with `moveTo / lineTo`. One code path covers 2-point (direct) and 3-point (mirrored) beams — no branching by beam type.

## What was removed from original plan

- Mirror surface angle / orientation in config — gone
- Deflection angle calculation in server — gone
- Any client-side angle math for outbound beam — gone
