# Session 2026-05-02

## Tasks
KNOWN_ISSUES audio pass: fix casting ability audio delay/no-start + remove Warlock/Druid DOT sounds.

## Root Cause Analysis

### Casting audio delay/no-start (systemic)
`syncPlayerState` in `AudioManager.js` had condition `castProgress > 0`. If the first `STATE_DELTA` arrives when the server computes `castProgress = 0` (elapsed ‚Čą 0 at tick boundary), the castSkill gets stored in `_channelPlayers` immediately. Every subsequent delta then has `previous.castSkill === castSkill`, so the audio trigger fires once ‚ÄĒ when progress was 0 and nothing played ‚ÄĒ and never fires again for that cast. Complete silence.

### Corruption silence during cast bar
Corruption (Warlock DOT) has `castBar: true` (800ms) but no `precast` defined in AudioConfig. The old `else if (skillAudio.precast)` branch never played anything for Corruption during the cast. The `cast` sound only fired at completion.

### DOT tick sounds (Corruption, Moonfire)
Every DOT tick emits `EFFECT_DAMAGE` with `sourceSkill: 'Corruption'` or `'Moonfire'`, which `handleEffectDamage` was converting to an impact sound on each tick.

## Changes Made

### `client/host/systems/AudioManager.js`

1. **Module-level constant** ‚ÄĒ `const DOT_SILENT_SKILLS = new Set(['Corruption', 'Moonfire'])`

2. **`handleEffectDamage`** ‚ÄĒ added early return: `if (DOT_SILENT_SKILLS.has(data.sourceSkill)) return`

3. **`syncPlayerState` trigger condition** ‚ÄĒ removed `castProgress > 0` check. New condition: `castSkill && castProgress != null && previous.castSkill !== castSkill`

4. **`syncPlayerState` cast-start audio** ‚ÄĒ replaced `else if (skillAudio.precast)` with unconditional `else` block that plays `precast ?? cast`. Abilities with precast: unchanged behavior. Abilities without precast (Corruption): plays cast sound at cast start.

## Behavior After Fix
- Corruption: plays `sfx_skill_corruption_cast` at cast start AND at cast completion (acceptable double-play separated by 800ms)
- Moonfire: DOT ticks silent; cast sound still plays on `SKILL_FIRED` ‚úď
- All cast-bar abilities: audio fires on first delta regardless of progress value ‚úď
