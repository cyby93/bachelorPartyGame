## What was done

### Task 6 — VFXManager registration system
**Modified:** `client/host/systems/VFXManager.js`

Replaced `triggerSkillVFX` switch/if-else double nesting with a two-tier map system:
- `_skillHandlers` Map (skillName → handler) — checked first, highest priority
- `_typeHandlers` Map (type → handler) — fallback for all generic types
- Added `registerSkillVFX(skillName, handler)` public method for external registration (Task 8 and future skills)
- `triggerSkillVFX` reduced to one line: `handler?.(data)`

Named handlers registered: Holy Nova, Frost Nova, Fear, Consecration, Bloodlust, Mass Resurrection, Tranquility.
Type handlers: MELEE, AOE, DASH, BUFF, SHIELD, CAST, CHANNEL, TARGETED, EXPLOSION, SPAWN.

### Task 7 — ProjectileSprite PROJECTILE_CONFIG
**Modified:** `client/host/entities/ProjectileSprite.js`

Collapsed four parallel maps (TRAIL_STYLE_MAP, SPIN_SPEED_MAP, BODY_SCALE_MAP, TRAIL_LENGTH_MAP) into a single `PROJECTILE_CONFIG` object keyed by spriteKey:
```js
const PROJECTILE_CONFIG = {
  'projectile_avengers_shield': { trailStyle: 'holy',   spinSpeed: 0.14 },
  'projectile_penance':         { trailStyle: 'divine', bodyScale: 0.88, trailLength: 8 },
  'projectile_ichor':           { trailStyle: 'ichor',  spinSpeed: 0.04, trailLength: 7 },
}
```
Constructor reads `cfg = PROJECTILE_CONFIG[data.spriteKey] ?? {}` once and destructures from it.
`trailLength` now lives on the sprite entry rather than on the trail style — more natural.
