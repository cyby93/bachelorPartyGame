## Changes Made

### `client/host/scenes/BattleRenderer.js` (tombstone revive arc)
- **Stray line bug fixed:** Added `gfx.moveTo(tomb.x, tomb.y)` before `gfx.arc(...)`. Root cause: after `gfx.fill(...)` the PixiJS path cursor resets to (0,0). The subsequent `arc()` call implicitly drew a line from (0,0) to the arc's start point — the stray green line from top-left corner. `moveTo` to the arc start point (tomb.x, tomb.y — derived from cx + r*cos(-π/2), cy + r*sin(-π/2)) eliminates it.
- **Circle thickness increased:** `gfx.stroke({ color: 0x00ff88, width: 3 })` → `width: 5`.

## Pattern to Remember
In PixiJS v8, always call `moveTo` to the arc start point before `arc()` when the prior operation was `fill()` or `stroke()` — those operations end the path and reset the cursor to (0,0), causing a stray connecting line on the next drawn shape.
